Checking the accessibility and usability of a document for low vision users in the OpenDocument Accessibility perspective

You can open an ODF document in the low vision view to simulate how low vision people (individuals with weak eyesight, color vision deficiencies, cataracts, or combinations of these vision problems) see an ODF document and to determine how easy or difficult the document is for them to read.

To check the accessibility and usability of a document for people with low vision:

  1. Open the ODF file in one of the following ways (OpenDocument accessibility perspective only supports local files):
    • Click File > Open, and then navigate to the local ODF file you want to check.
    • After selecting the Office view, click the Open ODF File button at the top of the Office view, and open the ODF file.
    • Type the URL in the address line, and then click GO.
      The document opens in the Office view.
  2. Click Low Vision above the top right pane to select the low vision view.
  3. (Optional) Select the whole page option if you want the tool to check the entire document.
    (Notes: The whole page function only supports presentation files (ODP files). If this check box is checked, the tool simulates all of the pages of a presentation document. If this check box is not checked, the tool simulates only the page currently shown in the Office view.)
  4. Open the Low Vision Simulation preference page in one of the following ways:
    • Click Setting Button.
    • Click Windows menu > Preferences > ACTF > Low Vision Simulation.
    Set the parameters for the three low vision types, and then click either Apply and Simulate or OK.
  5. Start the simulation and detect accessibility and usability problems with the parameters you have set in one of the following ways:
    • Select Simulate Button.
    • Select Window menu > Simulate (low vision) or use shortcut key (Ctrl+Shift+O).
    (Notes: In the OpenDocument accessibility perspective, Low vision simulation does not support the automatic problem detection in the Evaluation view. When examining the simulated image, please check that there are no images or characters that are difficult to read.)
  6. (Optional) Click Save to save the report file as an HTML document.
    (Notes: The report file is an HTML document, and it consists of three parts: a simulated image, a problem list, and a problem map. OpenDocument accessibility perspective supports only the simulated image.)
