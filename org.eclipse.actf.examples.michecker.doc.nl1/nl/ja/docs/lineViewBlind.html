詳細レポートの個別表示（音声ユーザビリティ）

  • 表示される情報
  • 操作

表示される情報

詳細レポートの個別表示には、miChecker で検出された問題点一つずつ列挙されます。 音声ユーザビリティ視覚化時には下記の情報が表示されます:

  • 問題点の種類: 問題点の種類を表すアイコンには、それぞれの以下のような意味があります:
    このアイコンは他のアイコンと併用されます。 このアイコンが表示されている行を選択すると、音声ユーザビリティビュー上でその問題点に対応する要素がハイライトされます。
    指摘箇所に問題が存在します。
    指摘箇所に問題が存在する可能性が高い項目です。
    評価すべき箇所をツールが指摘しますが、問題を含むかどうかは人が判断する必要がある項目です。
    ツールでは評価すべき箇所を指摘できない項目です。テスト方法にしたがって問題の有無を判断する必要があります。

    (注) 問題点が評価に影響を与える場合には、減点される数値がアイコンと共に表示されます。

  • 対応するアクセシビリティ・ガイドラインのチェック項目: 問題点に対応するアクセシビリティ・ガイドラインのチェック項目が表示されます。 使用するガイドラインは、以下に挙げる3種類のアクセシビリティ・ガイドラインや標準の中から自由に選択できます。 アクセシビリティ・ガイドラインを選択するには、メニューの 「ウィンドウ」 > 「設定」 から設定パネルを開き、「ACTF > ガイドラインチェッカー 設定」 でチェックボックスにチェックを入れます。
    • JIS X 8341-3
    • WCAG 2.0
    • Section 508
  • 達成方法: 対応する達成方法（実装方法）等の項目が表示されます。
  • 行番号: 対応する要素が存在する HTML 内の行番号が表示されます。
    (この項目はオプショナルです。メニューの 「ウィンドウ」 > 「設定」 から設定パネルを開き、「ACTF > ガイドラインチェッカー 設定」 の中の 「チェック対象」 で「HTMLファイル（行番号をエラーレポートに追加）を選択すると表示されます。)
  • 内容: 問題点や確認すべき内容が簡潔に記述されます。

操作

行を選択すると、対応する要素が音声ユーザビリティビュー上でハイライトされます。 右クリックするか、行が選択された状態で 「Shift+F10」 キーを押すと、コンテキストメニューが表示されます。 音声ユーザビリティ視覚化時には、コンテキストメニューから以下の5つの項目が利用できます。

  • 選択の解除: 個別表示における現在の選択が解除されます。これは、音声ユーザビリティ/ロービジョンビューにおけるハイライトを消す時に使用することができます。
  • 達成基準を見る: 選択されている項目に対応した達成基準などの項目を別ウィンドウに表示します。
  • 達成方法を見る: 選択されている項目に対応した達成方法（実装方法）などの項目を別ウィンドウに表示します。
  • エラーの内容を見る: 個別表示における「エラーの内容」の項目がポップアップ・ウィンドウに表示されます。「エラーの内容」が長すぎてmiCheckerの画面では表示しきれない場合に便利です。
  • ソースHTML を見る: HTMLソース ウィンドウを開きます。開いた HTMLソース ウィンドウ上で、選択されているエラーに該当する HTML 要素がハイライトされています。これにより、HTML 文書内のどこを直せばよいかがすぐにわかります。
  • コピー：　詳細レポートの内容をタブ区切り形式でクリップボードにコピーします。
  • プロパティ：　詳細レポートの内容をポップアップダイアログで確認できます。（各項目をダブルクリックしてもプロパティを確認することが出来ます。）
